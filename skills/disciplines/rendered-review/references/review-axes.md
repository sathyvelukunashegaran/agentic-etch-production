# Review axes

## Strategy and conversion

Does each page serve its named user situation and job? Can the user reach the intended action without losing context or trust?

## Copy and proof

Does the message hierarchy match the contract? Are claims specific, understandable and supported by their proof chains?

## Visual hierarchy and component grammar

Does composition reveal priority? Are repeated components coherent? Are variants intentional rather than accidental drift?

## Responsive behavior and accessibility

Check project-required widths, overflow, reading order, keyboard path, focus, landmarks, headings, labels, contrast, touch targets, reduced motion and error states.

## Stack integrity

Check Etch behavior, WordPress dynamic content, ACSS token use, scoped exceptions, console errors, network failures and editor-to-frontend consistency.

## Contract fidelity

List missing requirements, partial behavior, unapproved scope and implemented behavior that contradicts the approved contract.
