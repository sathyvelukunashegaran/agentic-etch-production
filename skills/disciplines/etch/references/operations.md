# Etch operation evidence

Before using an operation, prefer evidence in this order:

1. current project tooling and observed builder behavior;
2. installed version metadata;
3. official Etch documentation for that version;
4. existing working project patterns;
5. an isolated disposable experiment.

Record the evidence and any version constraint in `docs/agents/stack.md`.

Treat these as separate capability questions:

- page and template editing;
- reusable components and variants;
- dynamic WordPress data;
- forms and interactions;
- global styles and classes;
- responsive controls;
- import, export or code-based operations;
- preview and publishing behavior.

A verified capability in one category does not imply the others.
