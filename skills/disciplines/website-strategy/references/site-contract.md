# Site contract reference

For each important page capture:

- user and triggering situation;
- page job;
- entry conditions;
- primary conversion and fallback action;
- message hierarchy;
- proof obligations;
- content sequence;
- functional behavior;
- responsive and accessibility obligations;
- acceptance criteria;
- out-of-scope behavior.

Acceptance criteria should be observable in rendered behavior or content. Avoid implementation file paths unless the path itself is a project constraint.
