# Website Strategy

Use two leading artifacts:

- a **conversion path** is the complete journey from user intent to meaningful action;
- a **page contract** is the durable agreement for what one page must accomplish.

## Discipline

1. Start from the user's triggering situation, not a generic demographic.
2. Define one primary job for each page. Secondary jobs must not obscure it.
3. Connect every page to a conversion path or a necessary confidence-building role.
4. Separate business claims from evidence. Load `proof-chain` when claims shape strategy.
5. Prefer the smallest information architecture that serves the journeys.
6. Express strategy as observable obligations in `SITE-CONTRACT.md`, not abstract slogans.

Use [site-contract.md](references/site-contract.md) when drafting or reviewing the contract.

## Completion criterion

Strategy is sufficient when page existence, page order, message hierarchy and conversion choices can be justified from a named user situation and business outcome.
