# Shape mode

1. Confirm target surface, user, page job, content hierarchy, constraints, and existing system.
2. Write one sentence describing the physical use context: who uses it, where, under what light, with what urgency and mood.
3. Resolve typography character, color commitment, density, imagery, motion, interaction character, references, and anti-references.
4. Produce materially different directions. A palette swap is not a different direction.
5. For each direction explain composition, hierarchy, signature moves, responsive implications, risks, and why it fits the product.
6. Obtain explicit approval or delegation before implementation.
7. Record selected and rejected directions in `DESIGN.md`.

Keep a clear brief compact. Use a fuller exploration only when ambiguity, multiple surfaces, or stakeholders earn it.
