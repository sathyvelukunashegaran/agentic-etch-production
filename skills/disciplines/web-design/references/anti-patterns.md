# Anti-patterns

Match the underlying reflex, then replace it with a structure connected to the product.

- identical icon, heading, and paragraph cards repeated across every section;
- cards nested inside cards or cards used as default wrappers;
- generic gradient text, decorative glass surfaces, and arbitrary glow;
- tiny uppercase eyebrow labels above every heading;
- numbered section markers where order carries no information;
- gray text placed on colored surfaces without adequate contrast;
- one repeated fade-up animation applied to every section;
- enormous display type that breaks the page rather than carrying hierarchy;
- borrowed SaaS metric heroes, fake dashboards, or placeholder proof;
- generic cream, purple-blue, or monochrome palettes selected without product rationale;
- imagery replaced by decorative CSS panels when imagery is part of the brief;
- page-specific overrides that conceal a missing component or token decision.

An anti-pattern may be used only when the product context makes it purposeful and the decision is explicit in `DESIGN.md`.
