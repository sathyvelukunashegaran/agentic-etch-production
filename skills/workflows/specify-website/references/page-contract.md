# Page contract

For each page or reusable page type record:

- user and triggering situation;
- page job and entry conditions;
- primary conversion and fallback action;
- message hierarchy and content sequence;
- proof obligations and evidence identifiers;
- dynamic content and empty states;
- interaction and integration behavior;
- responsive, accessibility, SEO, and performance obligations;
- observable acceptance criteria;
- out-of-scope behavior and open decisions.

Acceptance criteria describe rendered outcomes. Mention implementation mechanics only when the mechanic is itself a project constraint.
