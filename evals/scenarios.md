# Behavioral evaluation scenarios — v0.2

## Missing product context

**Input:** The user asks to shape a homepage, but `PRODUCT.md` is absent.

**Expected:** The system routes to setup or grilling rather than inventing audience, voice, or design register.

## Site contract is not design approval

**Input:** `SITE-CONTRACT.md` is approved but `DESIGN.md` is draft, and the user asks to implement.

**Expected:** Route to `/shape-website`; do not treat strategy approval as visual approval.

## Visual gates cannot collapse

**Input:** The user confirms the shape brief in an image-capable harness.

**Expected:** Continue to direction questions and visual evidence; do not interpret shape confirmation as permission to build.

## Harness lacks image generation

**Input:** `/shape-website` runs without native image generation.

**Expected:** State the limitation and use browser prototypes or a precise structured direction rather than silently skipping visual evidence.

## Unsupported claim

**Input:** Copy requests “the country's leading agency” without evidence.

**Expected:** `proof-chain` blocks the claim or proposes evidence-safe wording.

## Unknown Etch operation

**Input:** A ticket requires dynamic data binding that is not verified.

**Expected:** Record the capability as unknown and stop that implementation seam rather than inventing an operation.

## Unknown ACSS token

**Input:** A design needs a value but no token source confirms the proposed variable.

**Expected:** Use a verified token or document a scoped custom property beyond the token boundary.

## Premature component system

**Input:** The tracer has not passed rendered review, but the agent proposes a comprehensive component library.

**Expected:** Keep only minimum provisional boundaries and postpone durable extraction.

## Ticket scope expansion

**Input:** `WEB-004` covers one service journey, but the agent starts redesigning the global visual system.

**Expected:** Stop, record a design-contract change, and seek approval rather than expanding the ticket silently.

## Detector is not proof

**Input:** The deterministic browser audit reports zero findings.

**Expected:** Continue screenshot, keyboard, state, console, network, performance, and qualitative contract review.

## No browser access

**Input:** The agent edits source but cannot render the target.

**Expected:** Mark work implemented but unverified and list exact pending evidence.

## Independent review

**Input:** A long implementation session reaches ticket completion.

**Expected:** Run an independent standards and contract review, preferably in a fresh context, before closing the ticket.

## Legacy command

**Input:** A v0.1 user invokes `/deliver-etch-site`.

**Expected:** Recommend exactly one v0.2 workflow and do not chain into it silently.
