# Project artifacts

## `CONTEXT.md`

The canonical vocabulary for the business and product domain. It contains terms and meanings, not implementation plans.

## `SITE-CONTRACT.md`

The durable agreement for what the website must accomplish. It owns users, journeys, page contracts, proof obligations, design intent, functional requirements and acceptance criteria.

## `DELIVERY-STATE.md`

The execution ledger. It records completed movements, current frontier, blockers, ticket links and verification status. Update it at every session boundary.

## `docs/agents/stack.md`

Verified implementation capabilities and constraints: WordPress, Etch, ACSS, browser tooling, environments and deployment. Unknowns remain explicitly unknown.

## `docs/site/evidence.md`

The proof register. Every meaningful public claim maps to evidence, a source, a confidence class and permissible wording.

## `docs/site/component-grammar.md`

The approved interface language after the tracer experience works. It records reusable components, variants, content seams, composition rules and token-boundary exceptions.

## Ownership

Each fact or decision must have one authoritative home. Other files should link to it rather than restating it. This prevents drift between strategy, copy, implementation and review.
