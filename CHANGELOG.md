# Changelog

## 0.2.0 — Design and delivery system

- decomposes the v0.1 monolithic delivery workflow into specification, shaping, planning, ticket implementation, review, and shipping workflows;
- adds `PRODUCT.md`, `DESIGN.md`, and `docs/site/delivery-plan.md` as durable project artifacts;
- deepens the web-design discipline with conditional registers, shape, craft, critique, state, responsive, visual-rule, and anti-pattern references;
- adds continuous rendered iteration and a deterministic browser audit;
- adds project migration and stage-aware artifact validation scripts;
- retains `/deliver-etch-site` as a compatibility router for v0.1 users;
- expands behavioral evaluation scenarios and beginner documentation.

## 0.1.0 — Foundation

- initial human-invoked workflows and model-invoked disciplines;
- project-memory templates, scaffolding, repository validation, and browser-truth principles.
